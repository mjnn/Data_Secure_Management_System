# 用户管理页 UI 设计说明

本文档描述 **控制台壳内**「用户管理」页面的信息架构、关键交互与视觉约定，与 **`docs/DSMS_PORTAL_UI_GUIDE.md`**（布局、令牌、玻璃面板、列表分页）及 **`docs/DSMS_IMPLEMENTATION_SPEC.md`**（用户目录、Excel 导入、项目成员、平台角色数据模型）保持一致。

---

## 1. 页面入口与壳层

- **路由**：作为 `DashboardPage` 内子路由（与 UI 指南一致：`el-main.dashboard-main` 内渲染），侧栏已有「用户管理」入口时指向本页。
- **顶栏**：不重复实现顶栏；复用现有 `dsms-portal-topbar`。
- **主区容器**：最外层使用 **`section.dsms-glass-panel`**（可加 `dsms-animate-stagger-*`），内边距与控制台主区一致。
- **页内标题**：`h2` 文案「用户管理」；`document.title` / `meta.title` 同步为「用户管理」（WCAG 2.4.2）。

---

## 2. 信息架构总览

采用 **「单页主从布局」**：上方 **工具栏 + 筛选**，中间 **用户主表**，下方或侧滑 **批量/详情抽屉**。避免在同一屏堆叠四个独立模块的四个大表单；将 **2～4** 类能力收敛为 **「主表选中态驱动的操作」**，降低认知负荷。

| 需求 | UI 承载方式 | 与规格/模型的对齐 |
|------|-------------|-------------------|
| 批量注册 | 工具栏主按钮「Excel 导入用户」+ 对话框（上传 + 结果摘要） | `POST .../platform/users/import-excel`、`GET .../template`（§6.1） |
| 批量注销 | 工具栏「批量停用」+ 二次确认（基于表格多选） | 语义对应 **`is_active=false`**；若后端暂无 PATCH 批量接口，实现阶段补 API，本设计按「批量提交 user_ids」预留 |
| 角色赋予/移除 | 单行「平台身份」或批量「设置平台角色」抽屉 | `user.platform_role`：`system_admin` \| `security_fo` \| `function_fo`（`models.User`）；**超级管理员**仍以 `is_superuser` 为准，UI 需区分展示，避免误改 |
| 项目分配 | 批量「加入项目」/「从项目移除」对话框；单行可展开项目标签 | `members/batch`、`members/batch-remove`（§6.3）；**升为 tenant_admin** 仅单行 + 单独确认，走 `PUT .../members/{user_id}/role` |
| 功能 FO 负责功能 | 仅当用户为 **`function_fo`** 时显示入口；抽屉内 **项目 → 空间 → 穿梭/双列表** | 业务 API 若在规格后续版本定义，前端契约与 OpenAPI 对齐；当前以 **占位接口说明** 支撑 UI 线框 |

## 3. 区域一：工具栏与筛选（主表之上）

**布局**：`el-row` / `flex` 一行内：**左侧筛选**，**右侧操作**（与现有门户「左辅右主」习惯一致）。

### 3.1 筛选（对齐 `GET /api/v1/dsms/users`）

- **关键词**：`el-input` 占位「用户名 / 邮箱 / 姓名」，防抖后刷新列表，`skip` 置 0。
- **账号状态**：`el-select` — 全部 / 仅启用 / 仅停用（映射 `is_active`）。
- **项目视角（可选）**：当前操作上下文若已选项目（与顶栏项目联动或本页内 `el-select` 项目）：`membership_preview_tenant_id` + `only_unassigned_to_tenant` 勾选「仅显示未加入该项目的用户」。
- **重置**：清空筛选并重新拉取。

### 3.2 主操作按钮组（从左到右优先级递减）

1. **Excel 导入用户**（`type="primary"`）：打开 §4 对话框。
2. **下载导入模板**（`type="primary"` `link` 或次要 `plain`）：直接请求模板下载。
3. **批量停用**：依赖表格多选；未选时 `disabled`；点击打开 §5.2。
4. **加入项目** / **从项目移除**：依赖多选；打开 §6 批量对话框。
5. **设置平台角色**（可选与「批量停用」同级）：多选打开 §5.1 抽屉。

**一致性**：主按钮深藏青、白字；危险操作用 `type="danger"` + `plain`，且必须二次确认。

---

## 4. 区域二：Excel 批量注册（对话框）

- **标题**：「批量导入用户（Excel）」。
- **内容区**：
  - 简短说明：必填列「邮箱」；可选「用户名」「姓名」「部门」；与规格列名映射一致。
  - `el-upload` **拖拽** + 文件类型限制 `.xlsx` / `.xls`。
  - 辅助链接：「下载标准模板」。
- **结果态**（同一对话框内步骤切换或底部 `el-result`）：
  - 展示 `created_count` / `skipped_count` / `total_rows`。
  - `skipped_items` 用 **`el-table`** 小表展示 `row` / `email` / `reason`（中文）。
- **操作**：「关闭」「继续导入」（清空文件回到上传步）。

---

## 5. 区域三：平台角色赋予与移除

### 5.1 设计原则

- **单值字段**：`platform_role` 在模型上为单字段，UI 使用 **单选**（`el-radio-group`）而非多选标签堆叠，避免与数据模型冲突。
- **与超级管理员关系**：若 `is_superuser === true`，显示 **只读徽章「超级管理员」**，`platform_role` 可灰显或固定说明「系统管理员账号」；批量「设置平台角色」应 **过滤或禁止** 包含超管行并 `ElMessage` 提示。
- **项目创建者名单**：与「角色」相关但 **API 为全量 PUT**（§6.1）。建议 **独立子区块** 或 **次级「平台权限」Tab**：
  - 「项目创建权名单」：`el-select` 多选用户（远程搜索 `q`）+ 保存时全量提交 `user_ids`；保存前 **确认对话框** 写明「将替换现有名单」。

### 5.2 批量停用（注销语义）

- **触发**：工具栏「批量停用」。
- **内容**：展示已选用户数；**不可选中当前登录用户**（前端校验 + 后端校验）。
- **确认**：`ElMessageBox.confirm` 文案使用「停用后不可登录」等明确后果。
- **成功后**：刷新列表，清空选择。

---

## 6. 区域四：用户可用项目分配

### 6.1 主表展示

- 列 **「项目与角色」**：`el-tag` 组展示 `项目名 · tenant_admin|tenant_member`；超出 2 个用「+N」+ 悬停 `el-popover` 完整列表（数据来自扩展接口或多次预览查询，实现时以 OpenAPI 为准）。
- 列 **「仅成员」筛选**与顶栏项目联动时，高亮在预览项目内「未加入」行（可选 `el-tag type="warning"`）。

### 6.2 批量加入项目

- **对话框标题**：「将选中用户加入项目」。
- **表单**：项目 `el-select`（`GET /tenants` 或现有列表源）；角色默认 **`tenant_member`**（`el-radio`）；说明文案引用规格：**不可**在本对话框批量设为 `tenant_admin`。
- **提交**：`members/batch`；返回中若有「跳过」项，用 `el-alert` + 可展开表格展示。

### 6.3 批量从项目移除

- **对话框**：选项目；警告文案：需保证项目内 **至少一名 tenant_admin**（规格 §6.3）。
- **提交**：`members/batch-remove`。

### 6.4 单行升为项目管理员

- 行操作「管理项目角色」→ 小抽屉或嵌套对话框：选项目 → 展示当前角色 → **「设为项目管理员」** 单独按钮 → `PUT .../role` + 强确认（防误触）。

---

## 7. 区域五：功能 FO 与负责功能（抽屉）

**可见性**：行内按钮「负责功能」仅当 **`platform_role === 'function_fo'`** 时启用；其他角色 `disabled` 或隐藏。

**抽屉结构（自上而下）**

1. **上下文**：`el-steps` 或面包屑：**项目** → **项目空间**（与业务 API 的 `{tenant_id}/{space_id}` 前缀一致）。
2. **功能列表**：
   - 左侧：**可选业务功能**（来自 `business-functions/options` 或等价列表，支持搜索）。
   - 右侧：**已绑定负责**列表。
   - 中间：`el-button` 组「添加」「移除」，或 **`el-transfer`**（注意深色模式下 Transfer 样式需在 `portal-theme` 下 spot-check）。
3. **批量**：支持在左右列表多选后一次移动；底部「保存绑定」主按钮、「取消」。

**空状态**：未选空间时右侧占位图 + 文案「请先选择空间」。

---

## 8. 主表列与分页（一致性）

| 列 | 说明 |
|----|------|
| 多选 | `type="selection"` |
| 用户名 | `username`，可链接打开「用户详情抽屉」（只读资料） |
| 邮箱 | `email` |
| 姓名 / 部门 | `full_name`, `department` |
| 平台身份 | `platform_role` 映射中文标签 + `el-tag` |
| 超级管理员 | 图标或 `el-tag`「超管」 |
| 项目 | 见 §6.1 |
| 状态 | `is_active` → 启用 / 停用 |
| 创建时间 | `created_at` |
| 操作 | 负责功能（FO）、项目角色、停用/启用（单行，若开放） |

**分页**：`el-pagination`：`layout="total, sizes, prev, pager, next"`；`page-size` 变更与筛选变更均 **重置 `skip` 为 0**；`limit` 默认 20、最大 500。

---

## 9. 视觉与组件约定（与门户一致）

- **颜色与边框**：卡片与表格容器使用 `var(--workspace-border)`、`var(--workspace-surface-2)`；勿引入新的主色体系。
- **间距**：筛选行与表格间距 **16～20px**；表格与分页 **12～16px**。
- **表格**：`el-table` 建议 `stripe`，表头背景与 `portal-theme` 中控制台表格一致；固定左列多选 + 右列操作。
- **抽屉宽度**：约 **480～640px**（角色/项目）；功能绑定抽屉 **720px** 以容纳 Transfer。
- **加载态**：表格 `v-loading`；上传 `el-progress`（若接口支持）或按钮 loading。

---

## 10. 权限与菜单

- **Excel 导入、项目创建者名单**：仅 **`super_admin`**（与 §6.1 一致）。
- **全局用户目录列表**：`super_admin` 或 **任一户下的 tenant_admin**（§6.2）；tenant_admin 在本页应 **隐藏** 无权限的按钮（导入、平台角色、创建者名单），仅保留 **检索用户 + 批量加入/移除自己管理的项目** 等能力（实现时在代码中集中判断）。
